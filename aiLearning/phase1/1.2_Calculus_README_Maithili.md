## 1. डेरिवेटिव (Derivatives)

### विचार
Derivative एकटा function क **बदलाव दर (rate of change)** बताबैत अछि, कोनो खास बिंदु पर — जँ input केँ थोड़ेक हिलाबी त output कतेक हिलत, आ कोन दिशामे? ज्यामितीय रूपसँ, ई ओहि रेखाक ढाल (slope) थिक जे ओहि बिंदु पर curve केँ केवल छूइत अछि (tangent line)।

### जुड़ल उदाहरण
अपन गाड़ीक speedometer के बारेमे सोचू। अहाँक position (कतेक मील चलल) समयक function थिक। कोनो पल पर अहाँक speed, position क time क सापेक्ष **derivative** थिक — मतलब "ई चीज अखन कतेक तेजीसँ बदलि रहल अछि।" जँ speed positive अछि, अहाँ आगाँ जाइत छी; शून्य त रुकल छी; negative त पाछाँ।

### ML मे ई कतए भेटत
- **Loss function** (मॉडल कतेक गलत अछि) एकटा curve थिक। ओहि curve क derivative बताबैत अछि जे कोनो खास weight बढ़ेला पर error बढ़त या घटत, आ लगभग कतेक।
- "Training" पूरा तरहेँ एहि एकहि विचार पर टिकल अछि।

### अभ्यास
1. **हाथसँ:** `f(x) = x²` लेल, derivative `f'(x) = 2x` अछि। `x = 1`, `x = 3`, आ `x = -2` पर slope निकालू। हर बिंदु पर sign की बताबैत अछि जे curve कोन दिशा जा रहल अछि?
2. **हाथसँ:** `f(x) = 3x + 5` (एकटा सीधा रेखा) लेल derivative सब जगह की अछि? ई constant किएक अछि से बुझू।
3. **NumPy — derivative के numerically अनुमान लगाउ (symbolic math नहि, बस definition):**
   ```python
   import numpy as np

   def f(x):
       return x**2

   def numerical_derivative(f, x, h=1e-5):
       return (f(x + h) - f(x)) / h   # slope between two very-close points

   for x in [1, 3, -2]:
       print(f"f'({x}) ≈ {numerical_derivative(f, x):.4f}  (true value: {2*x})")
   ```
4. **सोचू:** `h = 1` जकाँ बड़का value क बदला `h` केँ बहुत छोट (जेना 0.00001) किएक राखल जाइत अछि? कोनो घुमावदार (curvy) function पर बड़का `h` सँ की गड़बड़ी होएत?

## 2. आंशिक डेरिवेटिव (Partial Derivatives)

### विचार
असल ML मॉडलमे केवल एकटा input नहि रहैत अछि — हजारो या लाखो weights रहैत अछि। **Partial derivative** पूछैत अछि: "जँ हम केवल एहि एकटा input/weight केँ हिलाबी, आ बाकी सबकेँ स्थिर राखी, त output कोना बदलत?" ई हर weight लेल अलग-अलग कएल जाइत अछि।

### जुड़ल उदाहरण
कल्पना करू अहाँ केक बना रहल छी आ परिणाम (स्वादक अंक) oven क temperature आ baking time दुनू पर निर्भर करैत अछि। Temperature क सापेक्ष partial derivative पूछैत अछि: "जँ हम केवल temperature थोड़ेक बदली, time बिलकुल एकहि राखी, त स्वादक अंक कतेक बदलत?" Time लेल भी एकरे तरहक अलग सवाल पूछल जाइत अछि। हर सामग्री/knob के अपन अलग "संवेदनशीलता रिपोर्ट" भेटैत अछि।

### ML मे ई कतए भेटत
- एकटा न्यूरल नेटवर्कमे लाखो weights रहि सकैत अछि। Training के हर एकटा weight लेल loss क partial derivative चाही, ताकि ओकरा पता चलय जे हर एकटा केँ कोना समायोजित करी।
- सबटा partial derivatives (हर parameter लेल एक) क संग्रह केँ **gradient** कहल जाइत अछि — आगाँ देखल जाएत।

### अभ्यास
1. **हाथसँ:** `f(x, y) = x²y + 3y` लेल निकालू:
   - `∂f/∂x` (y केँ constant मानू)
   - `∂f/∂y` (x केँ constant मानू)
2. **हाथसँ:** दुनू partial derivatives केँ `x=2, y=1` पर निकालू।
3. **NumPy — numerical partial derivatives:**
   ```python
   def f(x, y):
       return x**2 * y + 3*y

   def partial_x(f, x, y, h=1e-5):
       return (f(x + h, y) - f(x, y)) / h

   def partial_y(f, x, y, h=1e-5):
       return (f(x, y + h) - f(x, y)) / h

   print("∂f/∂x at (2,1):", partial_x(f, 2, 1))
   print("∂f/∂y at (2,1):", partial_y(f, 2, 1))
   ```

## 3. ग्रेडिएंट (The Gradient)

### विचार
**Gradient** बस कोनो function क सबटा partial derivatives केँ एकठाम एकटा vector मे बान्हल रूप थिक। ई function क **सबसँ तेज बढ़तीक दिशा** दिस इशारा करैत अछि। ओकर sign उनटा दिय त **सबसँ तेज घटतीक दिशा** भेटत — जे error केँ न्यूनतम करबाक काल जरूरी होइत अछि।

### जुड़ल उदाहरण
अहाँ धुंधवाला पहाड़ीक ढलान पर ठाढ़ छी आ नीचाँ (सबसँ कम error) जायके चाहैत छी, मुदा दूर धरि नहि देखि पाबैत छी। Gradient जेना अहाँक पएरक नीचाँक जमीन महसूस करब अछि — ई बताबैत अछि अखन कोन दिशा "चढ़ाइ" थिक। नीचाँ जेबाक लेल, अहाँ gradient क **उनटा** दिशामे एक कदम बढ़ैत छी, फेर फेरसँ जमीन महसूस करैत छी (gradient फेरसँ निकालैत छी) आ दोहराबैत छी।

### ML मे ई कतए भेटत
- ई धुंधवाला-पहाड़ी वाला उदाहरण **गradient descent** थिक, जे लगभग हर ML/DL मॉडलक पाछाँक मुख्य training algorithm थिक।
- Gradient vector मे हर model parameter लेल एकटा entry रहैत अछि — दस लाख weights वाला नेटवर्क लेल gradient दस लाख-आयामी तीर थिक जे "नीचाँ" दिस इशारा करैत अछि।

### अभ्यास
1. **हाथसँ:** `f(x, y) = x² + y²` (एकटा कटोरा आकार) लेल gradient vector `∇f = [∂f/∂x, ∂f/∂y]` लिखू।
2. **हाथसँ:** बिंदु `(3, 4)` पर gradient निकालू। `f` केँ सबसँ तेज घटेबाक लेल कोन दिशा जाब चाही?
3. **NumPy — कटोरा पर gradient descent देखू:**
   ```python
   import numpy as np

   def f(x, y):
       return x**2 + y**2

   def grad(x, y):
       return np.array([2*x, 2*y])

   point = np.array([3.0, 4.0])
   lr = 0.1  # learning rate — how big a step to take

   for step in range(10):
       g = grad(point[0], point[1])
       point = point - lr * g   # move opposite to the gradient
       print(f"Step {step}: point={point}, f={f(point[0], point[1]):.4f}")
   ```
   देखू `f` कोना 0 दिस घटैत अछि जखन अहाँ "नीचाँ चलैत" छी।

## 4. चेन रूल (The Chain Rule)

### विचार
जखन एकटा function दोसर function क भीतर रहैत अछि — `f(g(x))` — त chain rule सँ पूरा चीजक derivative निकालल जाइत अछि, बाहरक function क derivative केँ भीतरक function क derivative सँ गुणा कऽ। ई "function क function" केँ differentiate करबाक तरीका थिक।

### जुड़ल उदाहरण
एकटा factory assembly line सोचू: कच्चा माल → Machine A आकार दैत अछि → Machine B रंग दैत अछि → अंतिम उत्पाद। जँ अहाँके जानब अछि "कच्चा मालमे थोड़ेक बदलाव सँ अंतिम उत्पादमे कतेक बदलाव होएत," त अहाँ chain मे हर machine क संवेदनशीलता गुणा करब: (B क output A क output मे प्रति इकाई बदलावसँ कतेक बदलैत अछि) × (A क output कच्चा मालक प्रति इकाई बदलावसँ कतेक बदलैत अछि)। पूरा chain मे संवेदनशीलता गुणा करू, त end-to-end संवेदनशीलता भेटत।

### ML मे ई कतए भेटत
- **ई backpropagation थिक।** न्यूरल नेटवर्क असलमे functions क chain थिक (layer 1 → layer 2 → layer 3 → loss)। ई जानबाक लेल जे layer 1 क एकटा weight अंतिम loss केँ कोना प्रभावित करैत अछि, अहाँ loss सँ ओ weight तक, layer-दर-layer derivatives केँ गुणा करैत छी। यैह पूरा backpropagation algorithm थिक — बार-बार chain rule, आर किछु नहि।

### अभ्यास
1. **हाथसँ:** `f(x) = (3x + 1)²` लिय। एकरा बाहरी function `h(u) = u²` मानू जतय `u = g(x) = 3x + 1`। `dh/du` निकालू, फेर `du/dx`, फेर दुनूकेँ गुणा कऽ `df/dx` निकालू। `(3x+1)²` केँ सीधे विस्तार कऽ differentiate कऽ जाँच करू — एकहि उत्तर भेटल?
2. **हाथसँ:** 2-step chain `y = sin(x²)` लेल, chain rule लागू करू: `dy/dx` की अछि?
3. **NumPy — numerically जाँच करू:**
   ```python
   import numpy as np

   def f(x):
       return (3*x + 1)**2

   def numerical_derivative(f, x, h=1e-5):
       return (f(x + h) - f(x)) / h

   x = 2.0
   analytical = 2 * (3*x + 1) * 3   # chain rule result: 2u * 3
   numerical = numerical_derivative(f, x)
   print("Analytical:", analytical, " Numerical:", numerical)
   ```
4. **हाथसँ छोट backprop:** मानू `loss = (prediction - actual)²`, आ `prediction = w * x` (एकटा छोट 1-weight linear model)। देल गेल `x=2, w=3, actual=10`:
   - `prediction` निकालू।
   - `loss` निकालू।
   - Chain rule उपयोग कऽ `d(loss)/d(w)` निकालू (संकेत: पहिने `d(loss)/d(prediction)`, फेर `d(prediction)/d(w)`, फेर गुणा करू)।
   - ई एकटा संख्या एहन थिक जे एकटा training step `w` केँ अपडेट करबामे उपयोग करत।

## 5. ग्रेडिएंट डिसेंट (सबटा बात मिलाकऽ)

### विचार
Gradient descent training loop थिक: हर parameter क सापेक्ष loss क gradient निकालू, ओकर उनटा दिशामे एकटा छोट कदम बढ़ाउ, आ दोहराउ जाबे धरि loss सुधरब बन्न नहि भऽ जाय।

### जुड़ल उदाहरण
धुंधवाला-पहाड़ीक चित्र जारी राखू: अपन पएरक नीचाँक ढलान महसूस करू (gradient), नीचाँ दिस सावधानीसँ एक कदम चलू (weights अपडेट करू), फेर नव जगहसँ फेरसँ ढलान महसूस करू, आ दोहराउ। हर कदमक आकार **learning rate** थिक। बहुत बड़ त अहाँ valley क तर पार करि जाइ सकैत छी (overshoot); बहुत छोट त नीचाँ पहुँचबामे बहुत समय लागत।

### ML मे ई कतए भेटत
- ई *ई* training algorithm थिक linear regression, logistic regression, आ आगाँक फेजमे बनाओल जायवाला हर न्यूरल नेटवर्क लेल (Phase 1.4 आ Phase 5 मे variants जेना SGD, Momentum, Adam ऊपर जोड़ल जायत)।

### अभ्यास
1. **From-scratch mini-project — बिना scikit-learn एकटा रेखा fit करू:**
   ```python
   import numpy as np

   # Fake data: y = 2x + 1, plus a little noise
   np.random.seed(0)
   X = np.linspace(0, 10, 50)
   y_true = 2 * X + 1
   y = y_true + np.random.normal(0, 1, size=X.shape)

   # Start with random guesses for slope (w) and intercept (b)
   w, b = 0.0, 0.0
   lr = 0.01

   for epoch in range(1000):
       y_pred = w * X + b
       error = y_pred - y

       # Partial derivatives of Mean Squared Error w.r.t. w and b
       dw = (2 / len(X)) * np.sum(error * X)
       db = (2 / len(X)) * np.sum(error)

       # Gradient descent update
       w -= lr * dw
       b -= lr * db

       if epoch % 100 == 0:
           loss = np.mean(error ** 2)
           print(f"Epoch {epoch}: w={w:.3f}, b={b:.3f}, loss={loss:.3f}")

   print(f"\nFinal: w={w:.3f}, b={b:.3f} (true values were w=2, b=1)")
   ```
2. **प्रयोग करू:** `lr = 0.5` आ फेर `lr = 0.0001` सँ फेरसँ चलाउ। हर स्थितिमे की होइत अछि? (एकमे overshoot/diverge होएत, दोसरमे बहुत धीमे-धीमे रेंगत।) जे देखल से लिखू।
3. **विस्तार करू:** हर epoch पर loss value केँ `matplotlib` सँ plot करू। ई "loss curve नीचाँ जाइत" ठीक ओहने अछि जे बाकी रोडमैपमे हर training log मे देखब — आब अहाँके पता अछि जे ई बस gradient descent अपन काज कऽ रहल अछि।

## 6. जैकोबियन आ हेसियन (Jacobians & Hessians) — Concept रूपमे, तुरंत जरूरी नहि

### विचार
- **Jacobian**: जखन एकटा function *बहुत* value output करैत अछि (केवल एक नहि), त Jacobian सबटा partial derivatives क पूरा तालिका थिक — हर output, हर input क सापेक्ष। ई gradient केँ multi-output functions लेल सामान्यीकृत करैत अछि।
- **Hessian**: "derivative क derivative" — second-order partial derivatives क एकटा matrix। ई loss surface क **curvature** (मोड़) क बारेमे बताबैत अछि (ई एकटा नरम कटोरा थिक कि एकटा तेज सँकरा घाटी?)।

### जुड़ल उदाहरण
जँ gradient बताबैत अछि पहाड़ी कोन दिशा ढलान अछि, त Hessian बताबैत अछि पहाड़ीक **आकार** — की ई चौड़ा नरम कटोरा थिक (नीचाँ जाएब आसान) या तेज सँकरा घाटी (overshoot आ divar सँ टकराएब आसान)? किछु उन्नत optimizer एहि आकारक जानकारी उपयोग कऽ चालाकीसँ कदम बढ़बैत अछि।

### ML मे ई कतए भेटत
- जखन एकटा layer मे बहुत output होइत अछि (जे हर असल layer मे होइत अछि) तखन backpropagation मे Jacobians आबैत अछि।
- Hessians एहि concept मे आबैत अछि जे किएक किछु loss surfaces optimize करब कठिन होइत अछि, आ किछु उन्नत optimization methods (Newton's method, natural gradient) मे — अहाँके एकरा रोज हाथसँ निकालबाक जरूरत नहि पड़त; deep learning frameworks (PyTorch/TensorFlow, Phase 5 मे अबैत अछि) **automatic differentiation** सँ ई करैत अछि।

### अभ्यास
1. **केवल Concept:** `f(x, y) = [x + y, x * y]` (2 outputs, 2 inputs) लेल, पूरा Jacobian मे कतेक partial derivatives रहत? (संकेत: outputs × inputs।)
2. **सोचू:** एकटा बहुत "घुमावदार" (high-curvature) loss surface training केँ एकटा smooth कटोरा-आकार surface सँ कम स्थिर किएक बना सकैत अछि, तइयो gradient क दिशा एकहि रहैत काल?

> **एकटा छोट "autograd" sanity-checker बनाउ, फेर एकरा उपयोग कऽ असल चीज train करू:**
> 1. `numerical_derivative()` function (single variable) आ `numerical_gradient()` function (multi-variable) दुनू from scratch लिखू, derivative क definition (छोट `h`, NumPy क अलावा कोनो library नहि) उपयोग कऽ।
> 2. `numerical_gradient()` उपयोग कऽ ऊपर linear regression exercise मे हाथसँ निकालल Mean Squared Error क analytical gradient formulas (`dw`, `db`) केँ फेरसँ जाँच करू। ई लगभग मिलक चाही।
> 3. Noisy dataset पर `y = wx + b` fit करबाक लेल पूरा gradient descent implement करू (scikit-learn बिना), जेना ऊपर।
> 4. Plot करू: (a) epochs पर loss curve, (b) scattered बिंदु पर fitted रेखा, (c) `w` आ `b` लेल loss surface क 3D या contour plot, जाहिमे अहाँक gradient descent path डॉट्स क रूपमे minimum दिस बढ़ैत देखाओल गेल हो।
