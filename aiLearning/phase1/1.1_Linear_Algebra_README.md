## 1. Vectors

### The idea
A vector is just a list of numbers, treated as *one thing* that has both a **size** and a **direction**. `[3, 4]` isn't "the number 3 and the number 4" — it's one arrow pointing from the origin to the point (3, 4).

### Relatable analogy
Think of giving someone directions: "walk 3 blocks east, 4 blocks north." You could describe that as two separate instructions, or as one arrow on a map. That arrow is a vector. Its length (5 blocks, via Pythagoras) is its **magnitude**; the way it's tilted is its **direction**.

### Where it shows up in ML
- A single row in your dataset (e.g., a house: `[sqft, bedrooms, age, price]`) is a vector.
- A word embedding (`"king"` → `[0.21, -0.4, 0.88, ...]`) is a vector — words that mean similar things point in similar directions.
- A gradient (which we'll hit in 1.2) is a vector telling you which direction to adjust your model's parameters.

### Exercises
1. **By hand:** Draw vectors `[2, 3]` and `[-1, 4]` on graph paper. Add them tip-to-tail. What point do you land on?
2. **By hand:** Compute the magnitude (length) of `[3, 4]` and of `[1, 1]`.
3. **NumPy:**
   ```python
   import numpy as np
   v1 = np.array([2, 3])
   v2 = np.array([-1, 4])
   print(v1 + v2)                 # vector addition
   print(np.linalg.norm(v1))      # magnitude
   ```
4. **Think about it:** If two customer-behavior vectors point in almost the same direction, what might that mean about the two customers? (This is the intuition behind recommendation systems.)

## 2. Matrices & Matrix Multiplication

### The idea
A matrix is a grid of numbers — rows and columns. Matrix multiplication is a way of applying a *transformation* to every vector in a dataset at once: rotating it, stretching it, squashing it, or mixing its features together.

### Relatable analogy
Imagine a recipe converter: you feed in "2 cups flour, 1 cup sugar" and it spits out "grams of flour, grams of sugar, calories." Each output is a weighted combination of the inputs. That's exactly what one row of a matrix multiplication does — it takes your input vector and computes weighted sums of it. A neural network layer is just this recipe converter with millions of "recipes" (weights) learned from data.

### Where it shows up in ML
- **A neural network layer** = `output = input @ Weights + bias`. That `@` is matrix multiplication.
- **A whole batch of images** processed at once on a GPU = one big matrix multiplication (this is *why* GPUs matter — they're built to do this in parallel).
- **Recommendation engines** (Netflix, Amazon) use matrix factorization: break a big "users × movies" matrix into two smaller matrices that, multiplied together, approximate people's tastes.

### Exercises
1. **By hand:** Multiply
   ```
   A = [[1, 2],        B = [[5, 6],
        [3, 4]]              [7, 8]]
   ```
   Compute `A @ B` row by row (row of A · column of B).
2. **By hand:** Try `B @ A`. Is it the same as `A @ B`? (This is why matrix multiplication is *not* commutative — order matters, unlike regular number multiplication.)
3. **NumPy — implement it yourself first, no shortcuts:**
   ```python
   import numpy as np

   def my_matmul(A, B):
       rows_A, cols_A = A.shape
       rows_B, cols_B = B.shape
       assert cols_A == rows_B, "Shape mismatch!"
       result = np.zeros((rows_A, cols_B))
       for i in range(rows_A):
           for j in range(cols_B):
               for k in range(cols_A):
                   result[i, j] += A[i, k] * B[k, j]
       return result

   A = np.array([[1, 2], [3, 4]])
   B = np.array([[5, 6], [7, 8]])
   print(my_matmul(A, B))
   print(A @ B)   # verify against NumPy's built-in
   ```
4. **Connect it:** Given a batch of 3 house vectors (sqft, bedrooms) and a weight matrix that converts them to (predicted_price), write out the matrix multiplication that would produce all 3 predictions in one shot.

## 3. Transpose

### The idea
Transposing a matrix flips it over its diagonal — rows become columns, columns become rows. `A[i][j]` becomes `A[j][i]`.

### Relatable analogy
Imagine a spreadsheet where students are rows and subjects are columns (scores). Transposing it gives you subjects as rows and students as columns — same data, different viewing angle. Useful when you need to "look at it from the other side."

### Where it shows up in ML
- Needed constantly to make matrix multiplication shapes line up correctly (a very common source of bugs when building neural nets).
- Used in computing gradients during backpropagation.

### Exercises
1. **By hand:** Transpose
   ```
   A = [[1, 2, 3],
        [4, 5, 6]]
   ```
2. **NumPy:**
   ```python
   A = np.array([[1, 2, 3], [4, 5, 6]])
   print(A.T)
   print(A.shape, A.T.shape)   # notice shape flips: (2,3) -> (3,2)
   ```
3. **Debug this:** You have `X` with shape `(100, 5)` (100 samples, 5 features) and weights `W` with shape `(5, 1)`. You want predictions of shape `(100, 1)`. What multiplication do you need — and would you ever need to transpose anything here? Now suppose instead `W` was accidentally given shape `(1, 5)` — how would you fix it using `.T`?

## 4. Dot Product & Cross Product

### The idea
- **Dot product**: multiply corresponding elements of two vectors and sum them up. Result is a single number. It tells you *how much two vectors point in the same direction*.
- **Cross product** (3D only): produces a new vector perpendicular to both inputs. Less used in day-to-day ML, but good to know conceptually.

### Relatable analogy
Dot product is like a **similarity score**. Imagine two friends each rating 5 movies from 1-10. If you multiply their ratings pair-by-pair and add it up, a high total roughly means "these two have similar taste." That's literally the intuition behind cosine similarity, which powers recommendation and search systems.

### Where it shows up in ML
- **Cosine similarity** (dot product, normalized) is how search engines and recommendation systems measure "how similar are these two things" (documents, products, user profiles).
- **Attention mechanisms in Transformers** (Phase 8 later in your roadmap) are built almost entirely on dot products — "how much should this word pay attention to that word" is computed with a dot product.

### Exercises
1. **By hand:** Compute the dot product of `[1, 2, 3]` and `[4, 5, 6]`.
2. **By hand:** Compute the dot product of `[1, 0]` and `[0, 1]`. What does a result of 0 tell you geometrically? (Hint: think about the angle between them.)
3. **NumPy:**
   ```python
   a = np.array([1, 2, 3])
   b = np.array([4, 5, 6])
   print(np.dot(a, b))

   # Cosine similarity from scratch
   def cosine_similarity(a, b):
       return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

   print(cosine_similarity(a, b))
   ```
4. **Practical:** Represent 3 movies as tiny "genre vectors" (e.g., `[action, romance, comedy]` scored 0-10 each). Compute cosine similarity between all pairs. Which two movies are most alike, according to the math?

## 5. Eigenvalues & Eigenvectors

### The idea
For a given matrix (transformation), most vectors change *both* direction and length when you multiply them by it. But a special few vectors only get **stretched or squashed** — they don't change direction at all. Those are **eigenvectors**, and the amount they get stretched by is the **eigenvalue**.

### Relatable analogy
Imagine stretching a rubber sheet with a picture of a compass drawn on it. Most arrows on the compass rotate and distort as you stretch the sheet. But there's usually one direction (say, exactly north-south) that just gets longer or shorter without bending. That direction is the eigenvector; how much longer it got is the eigenvalue.

### Where it shows up in ML
- **PCA (Principal Component Analysis)** — finds the eigenvectors of your data's covariance matrix. These eigenvectors are the directions of *maximum spread* in your data — i.e., the most "informative" directions, used for dimensionality reduction and visualization.
- **Understanding stability** in optimization (whether gradient descent explodes or converges smoothly relates to eigenvalues of certain matrices).

### Exercises
1. **By hand (conceptual):** For matrix `A = [[2, 0], [0, 3]]`, guess the eigenvectors without any calculation. (Hint: what happens to `[1,0]` and `[0,1]` when multiplied by A?)
2. **NumPy:**
   ```python
   A = np.array([[2, 0], [0, 3]])
   eigenvalues, eigenvectors = np.linalg.eig(A)
   print("Eigenvalues:", eigenvalues)
   print("Eigenvectors:\n", eigenvectors)
   ```
3. **Slightly harder:** Try `A = np.array([[4, 1], [2, 3]])` and repeat. Verify manually that `A @ eigenvector ≈ eigenvalue * eigenvector` for one of the pairs returned.

## 6. Matrix Decomposition (SVD & the PCA connection)

### The idea
**SVD (Singular Value Decomposition)** breaks *any* matrix `A` into three simpler matrices: `A = U · Σ · Vᵀ`. `Σ` (a diagonal matrix) contains "singular values" ranked by importance — the biggest ones capture most of the information in `A`. Throw away the small ones, and you've compressed the matrix while keeping most of its "signal."

**PCA** is essentially SVD applied to your data's covariance structure — it finds the smaller set of directions that explain most of the variation in your dataset.

### Relatable analogy
Think of packing for a trip. You could bring 50 items, but really 5 "outfit combinations" cover 90% of situations you'll encounter. SVD finds those "top combinations" (singular values/vectors) mathematically, ranked by how much they matter, so you can keep the top few and discard the rest with minimal loss.

### Where it shows up in ML
- **Image compression** — keep only the top-k singular values of an image matrix; get a much smaller file that still looks mostly right.
- **PCA for dimensionality reduction** — turn a 500-feature dataset into the 10 features that matter most, useful for visualization and speeding up downstream models.
- **Recommendation systems** — decomposing the giant user-item matrix into smaller "taste factor" matrices.

### Exercises
1. **Conceptual:** If an image is a 512×512 matrix of pixel brightness values, what do you think happens visually if you reconstruct it using only the top 5 singular values instead of all 512? Guess before you try it.
2. **NumPy — SVD-based image compression:**
   ```python
   import numpy as np
   import matplotlib.pyplot as plt
   from scipy import misc  # or load any grayscale image as a 2D array

   # img = your_grayscale_image_as_2d_numpy_array
   U, S, Vt = np.linalg.svd(img, full_matrices=False)

   k = 20  # try 5, 20, 50, 100 and compare
   compressed = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]

   plt.subplot(1, 2, 1); plt.imshow(img, cmap='gray'); plt.title("Original")
   plt.subplot(1, 2, 2); plt.imshow(compressed, cmap='gray'); plt.title(f"Compressed (k={k})")
   plt.show()
   ```
3. **NumPy — PCA from scratch (no sklearn):**
   ```python
   def my_pca(X, n_components):
       X_centered = X - X.mean(axis=0)             # center the data
       cov_matrix = np.cov(X_centered, rowvar=False) # covariance matrix
       eigvals, eigvecs = np.linalg.eig(cov_matrix)
       order = np.argsort(eigvals)[::-1]            # sort by importance
       top_vecs = eigvecs[:, order[:n_components]]
       return X_centered @ top_vecs                  # project onto top directions

   # Try it on any dataset with 4+ numeric columns, reduce to 2D, and plot it.
   ```
4. **Reflect:** After reducing a dataset to 2 principal components and plotting it, do points of the same category (if you have labels) cluster together? What does that tell you about how much "signal" was preserved?

## 7. Norms (L1 / L2)

### The idea
A **norm** measures the "size" of a vector.
- **L2 norm** (Euclidean): straight-line distance — `sqrt(sum of squares)`. The everyday notion of distance.
- **L1 norm** (Manhattan): sum of absolute values — like walking city blocks, only horizontal/vertical moves allowed, no diagonals.

### Relatable analogy
If you're walking through Manhattan, you can't cut diagonally through buildings — you walk blocks (L1 distance). A bird flying overhead takes the direct diagonal path (L2 distance). Same start and end point, different notions of "distance."

### Where it shows up in ML
- **Regularization**: L1 regularization (Lasso) tends to push some model weights to exactly zero (feature selection — "just don't use this feature at all"). L2 regularization (Ridge) shrinks weights smoothly without zeroing them out.
- **Loss functions**: Mean Squared Error is built on L2 distance; Mean Absolute Error is built on L1 distance. L1 is more robust to outliers.
- **k-NN and clustering algorithms** need a distance metric to decide "how close are these two points" — that's a norm.

### Exercises
1. **By hand:** For vector `[3, -4]`, compute the L1 norm and the L2 norm.
2. **NumPy:**
   ```python
   v = np.array([3, -4])
   l1 = np.linalg.norm(v, ord=1)
   l2 = np.linalg.norm(v, ord=2)
   print(l1, l2)
   ```
3. **Think about it:** You have a dataset with one wild outlier (a typo like `age = 999`). Would a model trained with L1 loss or L2 loss be more thrown off by it? Why? (Hint: squaring a huge error makes it *huge-er*.)
