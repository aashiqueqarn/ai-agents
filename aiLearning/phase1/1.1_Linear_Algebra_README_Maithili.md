## 1. सदिश (Vectors)

### विचार
सदिश (Vector) बस संख्याक एकटा सूची अछि, जकरा **एकटा वस्तुक रूपमे** देखल जाइत अछि जकर **आकार (magnitude)** आ **दिशा (direction)** दुनू होइत अछि। `[3, 4]` केवल "3 आ 4" नहि — ई origin सँ बिंदु (3, 4) तक जाइत एकटा तीर (arrow) अछि।

### जुड़ल उदाहरण
कोनो केँ रस्ता बतबैक कल्पना करू: "3 ब्लॉक पूब जाउ, 4 ब्लॉक उत्तर जाउ।" एकरा दू अलग निर्देशक रूपमे भी बता सकैत छी, या नक्शा पर एकटा तीरक रूपमे। ओ तीर एकटा सदिश अछि। ओकर लंबाई (5 ब्लॉक, पाइथागोरससँ) ओकर **magnitude** थिक; ओ जतय झुकल अछि से ओकर **direction** थिक।

### ML मे ई कतए भेटत
- अहाँक डेटासेटक एकटा row (जेना एकटा घर: `[sqft, bedrooms, age, price]`) एकटा vector थिक।
- एकटा word embedding (`"king"` → `[0.21, -0.4, 0.88, ...]`) एकटा vector थिक — मिलैत-जुलैत अर्थवाला शब्द एके तरहक दिशामे इशारा करैत अछि।
- Gradient (जकरा हम 1.2 मे देखब) एकटा vector थिक जे बताबैत अछि कोन दिशामे मॉडलक parameters बदलक चाही।

### अभ्यास
1. **हाथसँ:** ग्राफ पेपर पर `[2, 3]` आ `[-1, 4]` सदिश बनाउ। तिनका tip-to-tail जोड़ू। कोन बिंदु पर पहुँचब?
2. **हाथसँ:** `[3, 4]` आ `[1, 1]` क magnitude (लंबाई) निकालू।
3. **NumPy:**
   ```python
   import numpy as np
   v1 = np.array([2, 3])
   v2 = np.array([-1, 4])
   print(v1 + v2)                 # vector addition
   print(np.linalg.norm(v1))      # magnitude
   ```
4. **सोचू:** जँ दू ग्राहकक व्यवहार-सदिश लगभग एके दिशामे इशारा करैत अछि, त एकर मतलब ओ दुनू ग्राहकक बारेमे की भ' सकैत अछि? (ई recommendation system क पाछाँक मूल विचार थिक।)

## 2. मैट्रिक्स आ मैट्रिक्स गुणन (Matrix Multiplication)

### विचार
मैट्रिक्स (Matrix) संख्याक एकटा ग्रिड थिक — rows आ columns। मैट्रिक्स गुणन एकटा तरीका थिक जाहिसँ डेटासेटक हर vector पर एकही बेरमे एकटा **transformation** लागू कएल जाइत अछि: घुमाबय, खींचय, दबाबय, या features केँ मिलाबय।

### जुड़ल उदाहरण
एकटा recipe converter क कल्पना करू: अहाँ "2 कप आटा, 1 कप चीनी" देल छी आ ओ "ग्राम आटा, ग्राम चीनी, कैलोरी" निकालैत अछि। हर output input क weighted combination थिक। मैट्रिक्स गुणनक एक row एहने काज करैत अछि — ई अहाँक input vector लऽ क ओकर weighted sums निकालैत अछि। एकटा न्यूरल नेटवर्क लेयर बस ई recipe converter थिक, लाखो "recipes" (weights) सहित जे डेटासँ सीखल गेल अछि।

### ML मे ई कतए भेटत
- **न्यूरल नेटवर्कक एकटा लेयर** = `output = input @ Weights + bias`। ओ `@` मैट्रिक्स गुणन थिक।
- **इमेजक एकटा पूरा batch** एकही बेरमे GPU पर process होइत अछि — एकटा बड़ मैट्रिक्स गुणनक रूपमे (GPU एहि लेल जरूरी अछि कियाक ई एकरा parallel मे तेज करैत अछि)।
- **Recommendation engines** (Netflix, Amazon) matrix factorization उपयोग करैत अछि: एकटा बड़ "users × movies" मैट्रिक्स केँ दूटा छोट मैट्रिक्समे तोड़ल जाइत अछि जे गुणा करला पर लोगक पसंद के अनुमान लगबैत अछि।

### अभ्यास
1. **हाथसँ:** गुणा करू
   ```
   A = [[1, 2],        B = [[5, 6],
        [3, 4]]              [7, 8]]
   ```
   `A @ B` केँ row-by-row (A क row · B क column) गणना करू।
2. **हाथसँ:** `B @ A` कएल कऽ देखू। की ई `A @ B` सन्ने अछि? (एहि लेल मैट्रिक्स गुणन commutative नहि होइत अछि — सामान्य संख्याक गुणनसँ अलग, क्रम मायने रखैत अछि।)
3. **NumPy — पहिने खुद बनाउ, कोनो shortcut नहि:**
   ```python
   import numpy as np

   def my_matmul(A, B):
       rows_A, cols_A = A.shape
       rows_B, cols_B = B.shape
       assert cols_A == rows_B, "Shape mismatch!"
       result = np.zeros((rows_A, cols_B))
       for i in range(rows_A):
           for j in range(cols_B):
               for k in range(cols_A):
                   result[i, j] += A[i, k] * B[k, j]
       return result

   A = np.array([[1, 2], [3, 4]])
   B = np.array([[5, 6], [7, 8]])
   print(my_matmul(A, B))
   print(A @ B)   # verify against NumPy's built-in
   ```
4. **जोड़ू:** 3 घरक vector (sqft, bedrooms) आ एकटा weight matrix जे इनका (predicted_price) मे बदलैत अछि, से देल गेल अछि — ओ मैट्रिक्स गुणन लिखू जे एकही बेरमे सबटा 3 प्रेडिक्शन देत।

## 3. ट्रांसपोज (Transpose)

### विचार
मैट्रिक्सक transpose करला सँ ओकरा ओकर diagonal पर उनटा देल जाइत अछि — rows columns बनि जाइत अछि, columns rows बनि जाइत अछि। `A[i][j]` बनि जाइत अछि `A[j][i]`।

### जुड़ल उदाहरण
एकटा spreadsheet क कल्पना करू जाहिमे students rows होइथ आ subjects columns (marks)। Transpose करला पर subjects rows बनि जाइत अछि आ students columns — एकहि डेटा, बस देखबाक कोण बदलि जाइत अछि। जखन डेटा केँ "दोसर तरहसँ" देखबाक जरूरत हो त ई काज अबैत अछि।

### ML मे ई कतए भेटत
- Matrix multiplication क shapes केँ सही ढंगसँ मिलेबाक लेल हरदम एकर जरूरत पड़ैत अछि (न्यूरल नेटवर्क बनाबैत काल एकटा बहुत सामान्य bug क स्रोत)।
- Backpropagation मे gradients निकालबाक काजमे उपयोग होइत अछि।

### अभ्यास
1. **हाथसँ:** Transpose करू
   ```
   A = [[1, 2, 3],
        [4, 5, 6]]
   ```
2. **NumPy:**
   ```python
   A = np.array([[1, 2, 3], [4, 5, 6]])
   print(A.T)
   print(A.shape, A.T.shape)   # notice shape flips: (2,3) -> (3,2)
   ```
3. **Debug करू:** अहाँक लग `X` अछि shape `(100, 5)` (100 samples, 5 features) आ weights `W` shape `(5, 1)`। अहाँके `(100, 1)` shape क predictions चाही। कोन गुणन जरूरी अछि — आ की एहिठाम कहियो transpose क जरूरत पड़त? आब मानू `W` गलतीसँ shape `(1, 5)` भेटल — `.T` उपयोग कऽ एकरा कोना ठीक करब?

## 4. डॉट प्रोडक्ट आ क्रॉस प्रोडक्ट

### विचार
- **डॉट प्रोडक्ट (Dot Product)**: दूटा vector क संगत elements केँ गुणा कऽ जोड़ल जाइत अछि। परिणाम एकटा संख्या होइत अछि। ई बताबैत अछि दूटा vector **कतेक एके दिशामे इशारा करैत अछि**।
- **क्रॉस प्रोडक्ट** (केवल 3D): दुनू input सँ perpendicular एकटा नव vector बनाबैत अछि। रोजमर्राक ML मे कम उपयोग होइत अछि, मुदा concept रूपमे जानब नीक अछि।

### जुड़ल उदाहरण
डॉट प्रोडक्ट एकटा **similarity score** जकाँ अछि। कल्पना करू दू गोट दोस्त 5 फिल्मक rating 1-10 मे देलैथ। जँ अहाँ हुनकर ratings जोड़ी-जोड़ीमे गुणा कऽ जोड़ि दी, त एकटा उच्च कुल जोड़ लगभग एहि बात केँ बताबैत अछि कि "एहि दुनू के पसंद मिलैत-जुलैत अछि।" यैह intuition cosine similarity क पाछाँ अछि, जे recommendation आ search system केँ चलाबैत अछि।

### ML मे ई कतए भेटत
- **Cosine similarity** (normalized dot product) search engines आ recommendation systems मे "ई दुनू चीज कतेक मिलैत-जुलैत अछि" मापबाक तरीका थिक।
- **Transformers क Attention mechanism** (रोडमैपक Phase 8 मे बाद मे) लगभग पूरा dot product पर टिकल अछि — "एहि शब्दके ओहि शब्द पर कतेक ध्यान देबाक चाही" ई dot product सँ निकालल जाइत अछि।

### अभ्यास
1. **हाथसँ:** `[1, 2, 3]` आ `[4, 5, 6]` क dot product निकालू।
2. **हाथसँ:** `[1, 0]` आ `[0, 1]` क dot product निकालू। परिणाम 0 भेटला पर ज्यामितीय रूपसँ एकर की मतलब भेल? (संकेत: दुनूक बीचक कोणक बारेमे सोचू।)
3. **NumPy:**
   ```python
   a = np.array([1, 2, 3])
   b = np.array([4, 5, 6])
   print(np.dot(a, b))

   # Cosine similarity from scratch
   def cosine_similarity(a, b):
       return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

   print(cosine_similarity(a, b))
   ```
4. **व्यावहारिक:** 3 गोट फिल्मक "genre vector" बनाउ (जेना `[action, romance, comedy]`, हर एक 0-10 मे score कएल)। सबटा जोड़ीक बीच cosine similarity निकालू। गणितक हिसाबसँ कोन दू फिल्म सबसँ मिलैत-जुलैत अछि?

## 5. आइगेनवैल्यू आ आइगेनवेक्टर (Eigenvalues & Eigenvectors)

### विचार
कोनो देल गेल मैट्रिक्स (transformation) सँ जखन अहाँ कोनो vector केँ गुणा करैत छी, त अधिकांश vector क दिशा आ लंबाई दुनू बदलि जाइत अछि। मुदा किछु खास vector केवल **खिंचाइत या दबाइत** अछि — हुनकर दिशा नहि बदलैत। ओहि खास vector केँ कहल जाइत अछि **eigenvectors**, आ ओ कतेक खिंचैत अछि से **eigenvalue** थिक।

### जुड़ल उदाहरण
एकटा compass बनल रबर शीट खींचबाक कल्पना करू। अधिकांश तीर घुमि आ मुड़ि जाइत अछि जखन शीट खींचल जाइत अछि। मुदा प्रायः एकटा दिशा (जेना ठीक उत्तर-दक्षिण) एहन होइत अछि जे बिना मुड़नहि केवल लंबा या छोट भऽ जाइत अछि। ओ दिशा eigenvector थिक; ओ केतेक लंबा भेल से eigenvalue थिक।

### ML मे ई कतए भेटत
- **PCA (Principal Component Analysis)** — अहाँक डेटाक covariance matrix क eigenvectors निकालैत अछि। ई eigenvectors ओ दिशा होइत अछि जाहिमे डेटा **सबसँ बेसी फैलल** अछि — मतलब सबसँ जरूरी दिशा, जे dimensionality reduction आ visualization मे उपयोग होइत अछि।
- Optimization मे **stability** बुझबाक लेल (gradient descent फाटत या ठीकसँ converge करत, ई किछु मैट्रिक्सक eigenvalues सँ जुड़ल अछि)।

### अभ्यास
1. **हाथसँ (concept):** मैट्रिक्स `A = [[2, 0], [0, 3]]` लेल, बिना कोनो गणना कएने eigenvectors क अनुमान लगाउ। (संकेत: `[1,0]` आ `[0,1]` केँ A सँ गुणा करला पर की होइत अछि?)
2. **NumPy:**
   ```python
   A = np.array([[2, 0], [0, 3]])
   eigenvalues, eigenvectors = np.linalg.eig(A)
   print("Eigenvalues:", eigenvalues)
   print("Eigenvectors:\n", eigenvectors)
   ```
3. **थोड़ेक कठिन:** `A = np.array([[4, 1], [2, 3]])` लऽ क दोहराउ। हाथसँ verify करू जे `A @ eigenvector ≈ eigenvalue * eigenvector` कोनो एकटा pair लेल सही आबैत अछि की नहि।

## 6. मैट्रिक्स डिकम्पोजिशन (SVD आ PCA सँ जुड़ाव)

### विचार
**SVD (Singular Value Decomposition)** कोनो मैट्रिक्स `A` केँ तीन सरल मैट्रिक्समे तोड़ैत अछि: `A = U · Σ · Vᵀ`। `Σ` (एकटा diagonal matrix) मे "singular values" होइत अछि जे महत्वक क्रममे राखल रहैत अछि — सबसँ बड़का singular value सबसँ बेसी सूचना (information) रखैत अछि। छोट-छोट values हटा दियौ, त मैट्रिक्स compress भऽ जाइत अछि आ अधिकांश signal रहि जाइत अछि।

**PCA** असलमे SVD थिक जे अहाँक डेटाक covariance structure पर लागू कएल गेल हो — ई डेटाक ओहि छोट सेट दिशा खोजैत अछि जे सबसँ बेसी variation केँ बुझबैत अछि।

### जुड़ल उदाहरण
यात्रा लेल सामान बान्हबाक कल्पना करू। अहाँ 50 गोट चीज लऽ सकैत छी, मुदा असलमे 5 गोट "कपड़ाक combination" 90% स्थिति केँ संभालि लेत। SVD गणितीय रूपसँ ओहि "top combination" (singular values/vectors) केँ महत्वक क्रममे खोजैत अछि, ताकि अहाँ top किछु राखि सकी आ बाकी छोड़ि सकी, कम्तिये नुकसानसँ।

### ML मे ई कतए भेटत
- **इमेज compression** — इमेज मैट्रिक्सक केवल top-k singular values राखू; बहुत छोट फाइल भेटत जे लगभग एहने देखाइत रहत।
- **PCA dimensionality reduction लेल** — 500-feature डेटासेटके 10 गोट सबसँ जरूरी features मे बदलू, जे visualization आ आगाँक मॉडल तेज करबामे मददगार अछि।
- **Recommendation systems** — बड़का user-item matrix केँ छोटछोट "taste factor" matrices मे तोड़ब।

### अभ्यास

       eigvals, eigvecs = np.linalg.eig(cov_matrix)
       order = np.argsort(eigvals)[::-1]            # sort by importance
       top_vecs = eigvecs[:, order[:n_components]]
       return X_centered @ top_vecs                  # project onto top directions

## 7. नॉर्म्स (L1 / L2)

### विचार
**Norm** एकटा vector क "आकार" मापैत अछि।
- **L2 norm** (Euclidean): सीधा-रेखाक दूरी — `sqrt(sum of squares)`। रोजमर्राक "दूरी" वाला विचार।
- **L1 norm** (Manhattan): absolute values क जोड़ — जेना शहरक ब्लॉक-ब्लॉक चलब, केवल horizontal/vertical, कोनो diagonal नहि।

### जुड़ल उदाहरण
जँ अहाँ Manhattan मे चलैत छी, त diagonal मे बिल्डिंगक बीचसँ नहि जा सकैत छी — ब्लॉक-ब्लॉक चलैत छी (L1 distance)। ऊपरसँ उड़ैत चिड़ै सीधा diagonal रस्ता लेत (L2 distance)। एकहि start आ end बिंदु, मुदा "दूरी" क अलग विचार।

### ML मे ई कतए भेटत
- **Regularization**: L1 regularization (Lasso) प्रायः किछु weights केँ बिलकुल शून्य बना दैत अछि (feature selection — "ई feature बिलकुल उपयोग नहि करू")। L2 regularization (Ridge) weights केँ धीरे-धीरे छोट करैत अछि, शून्य नहि करैत।
- **Loss functions**: Mean Squared Error L2 distance पर आधारित अछि; Mean Absolute Error L1 distance पर। L1 outliers सँ कम प्रभावित होइत अछि।
- **k-NN आ clustering algorithms** केँ ई तय करबाक लेल distance metric चाही जे "ई दुनू बिंदु कतेक करीब अछि" — यैह norm थिक।

### अभ्यास
1. **हाथसँ:** vector `[3, -4]` लेल L1 norm आ L2 norm निकालू।
2. **NumPy:**
   ```python
   v = np.array([3, -4])
   l1 = np.linalg.norm(v, ord=1)
   l2 = np.linalg.norm(v, ord=2)
   print(l1, l2)
   ```
3. **सोचू:** अहाँक डेटासेटमे एकटा जबरदस्त outlier अछि (जेना टाइपोसँ `age = 999`)। L1 loss या L2 loss सँ प्रशिक्षित मॉडल एहिसँ बेसी प्रभावित होएत? किएक? (संकेत: बड़ error केँ square करला पर ओ **आर बेसी** बड़ भऽ जाइत अछि।)
